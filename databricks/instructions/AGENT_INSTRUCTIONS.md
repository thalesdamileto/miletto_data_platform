# Metadata Template Rules

All pipeline metadata files generated by the AI agent must follow the
same structure as the JSON files located in the directory:

pipelines_metadata/

These files define the canonical metadata schema used by the pipeline framework.

The AI agent MUST treat these files as the source of truth.

---

# Template Source

Before generating any metadata file, the agent must inspect existing
metadata files located in:

pipelines_metadata/

The agent must select the closest matching example and use it as a template.

The agent must NOT invent new metadata structures.

---

# Metadata Generation Strategy

When creating a new pipeline metadata file, the agent must:

1. Search for existing files in `pipelines_metadata/`
2. Identify the most similar pipeline
3. Copy the structure of that file
4. Modify only the necessary fields

The agent must prioritize **consistency with existing metadata files** over creating new structures.

---

# Strict Structure Rule

The generated JSON must preserve:

- key names
- nesting structure
- optional sections
- formatting patterns
- array structure

The agent must not remove fields that appear in other metadata files
unless the example template also omits them.

The agent must never introduce new keys that do not exist in the
existing metadata templates.

---

# Bronze Pipeline Template Rules

Bronze pipelines using Databricks Volumes must follow the template located in:

pipelines_metadata/bronze/databricks_volume/json/

Example template file:

pipelines_metadata/bronze/databricks_volume/json/dlt_volume_to_json.json

These pipelines ingest raw data from Databricks Volumes into the bronze catalog.

---

## Fields That May Change (Bronze)

When generating a bronze pipeline the agent may modify:

- `name`
- `configuration.source_path`
- `configuration.target_schema`
- `configuration.target_table`
- `configuration.description`
- `root_path`

---

## Fields That Must Not Change (Bronze)

The following fields must remain identical to the template:

- `pipeline_type`
- `libraries`
- `continuous`
- `development`
- `photon`
- `channel`
- `catalog`
- `serverless`

The notebook path defined in:

libraries.glob.include

must not be modified.

---

## Bronze Naming Rules

Pipeline name format:

dlt_volume_to_json-bronze.<schema>.<table>

Example:

dlt_volume_to_json-bronze.dbo.customers

---

## Bronze Source Path Rule

Source paths must follow the Databricks Volume format:

/Volumes/workspace/default/raw/<table>/

Example:

/Volumes/workspace/default/raw/customers/

---

## Bronze Root Path Rule

Root paths must follow this pattern:

/Workspace/Shared/rooth_path/bronze/<schema>/<table>

Example:

/Workspace/Shared/rooth_path/bronze/dbo/customers

---

## Bronze File Location

New bronze metadata files must be created in:

pipelines_metadata/bronze/databricks_volume/json/

File naming rule:

dlt_volume_to_json_<table>.json

Example:

pipelines_metadata/bronze/databricks_volume/json/dlt_volume_to_json_customers.json

---

# Silver Pipeline Template Rules

Silver pipelines must follow the template located in:

pipelines_metadata/silver/

Example template file:

pipelines_metadata/silver/contracts.json

Silver pipelines define **data contracts and transformations from bronze tables to curated datasets**.

The structure defined in `contracts.json` must be used as the reference template.

The AI agent must always copy this structure when creating new silver pipelines.

---

## Fields That May Change (Silver)

When generating a silver contract the agent may modify:

- table name
- source bronze table
- target table
- column definitions
- transformation logic
- descriptions

All other fields must remain consistent with the contract template.

---

## Fields That Must Not Change (Silver)

The agent must not:

- introduce new structural keys
- remove required sections
- change contract structure
- modify unrelated contracts

Only the specific table contract being created should be modified.

---

## Silver File Location

Silver metadata contracts must be created or updated in:

pipelines_metadata/silver/

Example:

pipelines_metadata/silver/contracts.json

If the contract file already exists, the agent must **append the new contract definition** rather than replacing the file.

---

# File Location Rule

New pipeline metadata files must be created in:

pipelines_metadata/

Bronze pipeline metadata:

pipelines_metadata/bronze/databricks_volume/json/

Silver contracts:

pipelines_metadata/silver/

---

# Example Generation Workflow

When generating a new pipeline, the agent should:

1. Inspect the folder:

pipelines_metadata/

2. Copy the structure from the closest existing template.

3. Modify only the fields required for the requested pipeline.

4. Save the new metadata file following the naming conventions.

---

# Safety Rules

The agent must never:

- modify pipeline notebooks
- modify existing pipeline templates
- modify unrelated repository files
- invent new metadata schemas

All pipeline creation must happen exclusively through metadata JSON files.
